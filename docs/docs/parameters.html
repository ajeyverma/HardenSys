Note: All parameters are based on CIS benchmarks and security best practices for Windows systems.

Overview

HardenSys covers 121 security parameters across 7 main categories, providing comprehensive coverage of Windows security hardening requirements. Each parameter includes detailed configuration instructions and compliance checking.

Parameter Structure

Each security parameter includes:

  • Title: Descriptive name of the security setting
  • Details: Detailed description and recommended configuration
  • Script Key: Internal function identifier for automation
  • Category: Security category classification
  • Subcategory: Specific subcategory within the main category

Manual Setup

For environments where automated configuration is not possible, HardenSys provides detailed manual setup instructions for each parameter.

Setup Methods

Local Security Policy

Configure parameters using the Local Security Policy editor (secpol.msc)

  1. Open Local Security Policy
  2. Navigate to the appropriate category
  3. Configure the parameter as specified
  4. Apply and verify the setting

Registry Editor

Configure parameters directly in the Windows Registry

  1. Open Registry Editor (regedit)
  2. Navigate to the specified registry path
  3. Create or modify the registry value
  4. Restart the system if required

Command Line

Configure parameters using command-line tools

  1. Open Command Prompt as Administrator
  2. Run the specified command
  3. Verify the configuration
  4. Test the setting

Group Policy

Configure parameters using Group Policy Management

  1. Open Group Policy Management Console
  2. Create or edit a Group Policy Object
  3. Navigate to the appropriate policy section
  4. Configure and deploy the policy

Verification Procedures

After manual configuration, verify that the settings have been applied correctly:

Command Line 
# Verify password policy settings
net accounts

# Verify security options
secedit /export /cfg security_policy.cfg

# Verify registry settings
reg query "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System"

Best Practices

Important: Always create a backup of your current security policy before making changes.
  • Test changes in a non-production environment first
  • Document all manual changes for audit purposes
  • Verify settings after system restarts
  • Monitor for any unexpected behavior
  • Keep detailed records of configuration changes
